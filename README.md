#  FoodBridge – Surplus Food Marketplace

FoodBridge is a hyperlocal platform that connects people with surplus food to nearby users who can purchase it at affordable prices. It aims to reduce food waste while improving access to low-cost meals.

---

## Problem Statement

A significant amount of food is wasted at the consumer level due to the lack of real-time redistribution systems. At the same time, students and individuals often struggle to find affordable food options.

There is no efficient platform that connects surplus food providers with nearby users in a timely and accessible manner.

---

##  Solution

FoodBridge provides a simple, real-time platform where users can:

- List surplus food with details
- Discover nearby food listings
- Access affordable meals easily
- Reduce food waste

The system focuses on **hyperlocal discovery**, **simplicity**, and **fast interaction**.

---

##  Features

### Add Food Listing
- Enter food name, quantity, and price
- Upload food image
- Set expiry time

### Map-Based Location Selection
- Integrated map using OpenStreetMap
- Click to select exact pickup location
- Automatic address detection

### Safety Score
- Calculates food safety based on expiry time
- Helps users judge freshness

### Food Listings
- View all available food items
- Displays:
  - Name
  - Image
  - Expiry
  - Safety score

###  Local Storage
- Stores food listings locally
- Instant updates without backend

---

##  Tech Stack

### Frontend
- React (TypeScript)
- Tailwind CSS

### Maps
- Leaflet.js
- OpenStreetMap API (free)

### Storage
- Browser LocalStorage

---

##  How It Works

1. User adds surplus food using the form
2. Selects pickup location using the map
3. Uploads image and sets expiry
4. Listing is stored and displayed instantly
5. Other users can browse available food

---

##  Key Highlights

- Hyperlocal food discovery
- Real-time listing updates
- Simple and user-friendly interface
- Map-based location selection
- No external backend required

---

## Product Thinking

We intentionally designed a **pickup-based model** to:

- Reduce complexity
- Ensure faster transactions
- Improve scalability

The focus is on solving the core problem of **food discovery and redistribution first**, before adding logistics.
